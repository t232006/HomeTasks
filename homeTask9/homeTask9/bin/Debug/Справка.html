Введение.

Программа предназначена для создания и решения тестов, предполагающих ответ «да» или «нет». Для перехода между этими режимами следует использовать соответствующий пункт горизонтального меню.

Пункт меню «Новый...» предназначен для создания новой базы данных вопросов, либо для нового тестирования.

Режим тестирования.

После запуска программа сразу же открыта в режим тестирования, но для начала работы необходимо загрузить базу данных теста. Сделать это можно либо воспользовавшись пунктом меню «Открыть», либо нажатием сочетания клавиш Ctrl+O. Программа работает с форматом файлов .xml. После загрузки базы данных тестов программа сама выберет из предложенной базы вопросов то количество, которое указано в настройках программы («режим правки»).


Рисунок 1






  1. Заключение. Появляется только после завершения теста. Будет указано количество правильных ответов, процент правильных ответов и необходимый процент. Если тест не сдан, то надпись будет красная, если тест сдан — то зеленая.

  2. Окно содержания вопроса.

  3. Список предложенных вопросов.

  4. Кнопки навигации. В ходе решения теста есть возможность вернуться к любому вопросу и изменить свой ответ, либо отложить ответ на вопрос.

  5. Кнопка «Закончить». Появляется только тогда, когда даны ответы на все вопросы (статусная строка заполнена).

  6. Маркеры ответов. Когда тест закончен, маркеры становятся неактивными. При даче ответа, тест автоматически перемещается на другой вопрос.

  7. Статусная строка. Заполняется по мере ответов на вопросы теста.

Режим правки.


Рисунок 2




  1. Кнопка «добавить вопрос». При ее нажатии включается режим добавления вопроса (см. рисунок 3).

  2. Кнопка «удалить вопрос». При ее нажатии выбранный вопрос удаляется (если вопрос не выбран — появится сообщение об ошибке). При удалении вопроса появится запрос подтверждения:


Рисунок 3




  1. Кнопка «редактировать вопрос». Включается режим редактирования вопроса. Содержание вопроса можно будет изменить.

  2. Флаг «ответ». Если ответ «да», то следует поставить флаг, иначе оставить флаг пустым.

  3. Количество вопросов. Можно указать, сколько вопросов необходимо иметь в тесте.

  4. Порог сдачи. Указать, сколько в процентном соотношении правильных ответов необходимо для сдачи теста.


    Рисунок 4


В режиме добавления вопроса кнопки «добавить вопрос» и «удалить вопрос» окрасятся в зеленый цвет. Нужно составить сам вопрос и еще раз нажать на кнопку «добавить вопрос», если вопрос устраивает, либо отменить добавление вопроса нажатием кнопки «удалить вопрос».

Имейте ввиду, что, изменив содержание или перечень вопросов, необходимо сделать сохранение в базе данных. Это можно сделать либо при закрытии программы, либо выбором пункта меню «Сохранить...», либо нажав сочетание клавиш Ctrl+S.
